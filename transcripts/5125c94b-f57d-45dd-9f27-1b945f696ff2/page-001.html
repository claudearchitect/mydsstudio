Diagnosis

  • Vercel deploys from your GitHub remote (origin/main), not your local machine.
  • origin/main is stuck at commit a18be6c "Implementation and V0 Plan" — that commit predates the app scaffold. The remote has no package.json with next in it (I confirmed: git show origin/main:package.json has no next).
  • Your local main is 15 commits ahead, and the very first of those unpushed commits (918634c Phase 0: scaffold + frozen contracts) is what introduced package.json, next.config.ts, and the whole app.

So Vercel is correctly reporting that the code it can see has no Next.js. The Root Directory setting is a red herring — package.json is already at the repo root.
